PROMPT 2: Cung cấp SRS và yêu cầu sinh project structure
--------------------------------------------------
Tuyệt vời! Dưới đây là yêu cầu từ SRS (Software Requirements Specification)
của dự án eKYC - ABC Bank:

[YÊU CẦU NGHIỆP VỤ]
Tính năng: Đăng ký mở tài khoản ngân hàng cơ bản

Input (Request):
- fullName: String (bắt buộc, 2-100 ký tự)
- phone: String (bắt buộc, đầu số VN: 03x/07x/08x/09x, 10 số)
- email: String (bắt buộc, định dạng email hợp lệ)
- citizenId: String (bắt buộc, đúng 12 chữ số số)

Output (Response):
- accountId: UUID
- accountNumber: String (16 số, unique)
- status: Enum (PENDING / ACTIVE)
- createdAt: LocalDateTime

[YÊU CẦU KỸ THUẬT]
1. Java 17, Spring Boot 3.2.x
2. Spring Data JPA + PostgreSQL
3. Bean Validation (@Valid, @NotBlank, custom validator)
4. Global Exception Handling (@ControllerAdvice)
5. Response wrapper (ApiResponse<T> generic)

Hãy sinh đầy đủ cấu trúc thư mục (project structure) trước khi viết code.
Format: Hiển thị dạng cây thư mục.

PROMPT 3: Sinh Entity và DTO classes
--------------------------------------------------
Tốt! Bây giờ hãy sinh code cho các class sau, theo thứ tự:

1. **CustomerEntity.java** - JPA Entity
   - Bảng: customers
   - Các trường: id (UUID), fullName, phone, email, citizenId, accountId, accountNumber, status, createdAt, updatedAt
   - Annotations: @Entity, @Table, @Id, @GeneratedValue, @Column, @CreationTimestamp, @UpdateTimestamp
   - Phải có JavaDoc giải thích mục đích class và từng trường quan trọng

2. **AccountStatus.java** - Enum
   - Các giá trị: PENDING, ACTIVE, REJECTED, SUSPENDED
   - Có mô tả (description) cho từng giá trị

3. **RegistrationRequest.java** - DTO (Request)
   - Có Bean Validation annotations
   - Custom validator cho citizenId (phải đúng 12 số)
   - @NotBlank, @Email, @Pattern với regex cho phone VN

4. **RegistrationResponse.java** - DTO (Response)
   - Dùng Java 17 Record nếu có thể

5. **ApiResponse.java** - Generic Response Wrapper
   - Generic type T
   - Các field: success (boolean), message, data (T), timestamp
   - Static factory methods: success(data), error(message)

Mỗi class phải có đầy đủ JavaDoc và comment tiếng Việt cho logic phức tạp.

PROMPT 4: Sinh Repository và Service
--------------------------------------------------
Tiếp tục sinh:

6. **CustomerRepository.java** - Spring Data JPA Repository
   - Extends JpaRepository<CustomerEntity, UUID>
   - Custom queries: findByPhone, findByCitizenId, existsByPhoneOrCitizenId
   - Dùng @Query nếu cần query phức tạp

7. **RegistrationService.java** - Service Interface
   - Method: register(RegistrationRequest request): RegistrationResponse
   - Mô tả trong JavaDoc: pre-conditions, post-conditions, throws

8. **RegistrationServiceImpl.java** - Service Implementation
   - Implement đầy đủ logic:
     a) Kiểm tra trùng phone/citizenId
     b) Validate CCCD còn hiệu lực (giả lập)
     c) Generate accountNumber duy nhất (16 số)
     d) Tạo customer mới với status PENDING hoặc ACTIVE
     e) Lưu vào database
   - Comment tiếng Việt giải thích từng bước logic
   - Ném BusinessException nếu vi phạm business rule

--------------------------------------------------
PROMPT 5: Sinh Controller và Exception Handling
--------------------------------------------------
Tiếp tục:

9. **RegistrationController.java** - REST Controller
   - @RestController, @RequestMapping("/api/v1/registration")
   - Method: POST /register với @Valid, @RequestBody
   - Trả về ResponseEntity<ApiResponse<RegistrationResponse>>
   - Có @Operation annotation (Swagger/OpenAPI)
   - JavaDoc mô tả endpoint

10. **GlobalExceptionHandler.java** - @ControllerAdvice
    - Xử lý các exception:
      * MethodArgumentNotValidException → 400 Bad Request với danh sách lỗi validation
      * BusinessException → 422 Unprocessable Entity
      * DataIntegrityViolationException → 409 Conflict
      * Exception (fallback) → 500 Internal Server Error
    - Response format nhất quán với ApiResponse

11. **BusinessException.java** - Custom Exception
    - Kế thừa RuntimeException
    - Có errorCode (String) và message
    - Enum ErrorCode cho các loại lỗi: PHONE_ALREADY_EXISTS, CITIZEN_ID_ALREADY_EXISTS, INVALID_CITIZEN_ID

--------------------------------------------------
PROMPT 6: Sinh Custom Validator
--------------------------------------------------
Tạo thêm:

12. **CitizenIdValidator.java** - Custom Bean Validation Constraint
    - Annotation: @ValidCitizenId
    - Validate: đúng 12 số; không phải "000000000000"
    - Có message tiếng Việt: "Số CCCD phải gồm đúng 12 chữ số"

13. **VietnamesePhoneValidator.java** - Custom Validator cho SĐT VN
    - Validate đầu số hợp lệ: 03x, 07x, 08x, 09x
    - Annotation: @ValidVietnamesePhone
